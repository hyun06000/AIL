# R6 Benchmark Analysis — ail-coder:7b-v6 (non-coder base + single-line training)

**Date:** 2026-04-22
**Model:** ail-coder:7b-v6 (qwen2.5-7b-instruct non-coder base + QLoRA r=16, 291 samples, 3 epochs, single-line chatml)
**Backend:** vLLM on homeblack, Q4_K_M GGUF via peft-merge path
**Condition:** C4 (ft/nofs)
**Result file:** `2026-04-22_r6_cond4_noncoder_finetuned_nofewshot.json`

## Verdict

**Hypothesis from R5 analysis CONFIRMED.** The v5 single-line regression was caused by the coder base model's Python pretraining prior, not by the single-line format itself. v6 uses a non-coder base with identical training data and identical single-line format, and **no longer exhibits the leading-quote artifact**. Cat C parse recovers from 20% → 70%.

v6 still trails v3 by 8pp overall (62% vs 70% answer correctness), but the Cat C recovery is dramatic and the grammar's error-handling guarantee holds at 0% as expected. v3 remains the serving model (indented + coder), but v6 establishes that single-line training is a viable path given the right base.

## Headline — v6 vs v5 vs v3

| Metric | v6 (non-coder + single-line) | v5 (coder + single-line) | v3 (coder + indented) |
|---|---|---|---|
| AIL parse | **80%** | 50% | 80% |
| AIL answer | **62%** | 42% | 70% |
| fn/intent routing | 72% | 48% | — |
| Leading-quote artifact in output | **0/50** | **23/50** | — |
| avg completion tokens | 83 | 59 | 89 |
| AIL error-handling omission | 0% | 0% | 0% |

The leading-quote artifact row is the smoking gun. v5 had 23/50 programs starting with `"pure fn ...` (parsed as string literal, rejected). v6 has zero. **Same training data, same format, different base model — the artifact disappeared.**

## Per-category breakdown — where the recovery happened

| Category | v3 ans | v5 ans | v6 ans | Δ vs v5 |
|---|---|---|---|---|
| A — pure computation (15) | 60% | 60% | 67% | +7pp |
| B — pure judgment (15) | 80% | 53% | 53% | = |
| C — hybrid (20) | 70% | **20%** | **65%** | **+45pp** |

Cat C was where v5 collapsed the hardest (−50pp from v3). v6 recovers +45pp of that, landing just −5pp short of v3. Cat A is actually better than v3 under v6, which is consistent with the reinforcement samples in `10_cat_b_reinforcement.jsonl` carrying some fn-heavy examples.

Cat B stayed flat. v5 and v6 both get 53% answer with 73-87% parse. The non-coder base didn't fix the `intent` silent-skip / wrong-goal issues that plagued v5 on Cat B — so those are separate problems from the coder-prior failure mode.

## What this says about the base model

v6's base (`Qwen/Qwen2.5-7B-Instruct`, general instruction-tuned) vs v5's base (`Qwen2.5-Coder-7B-Instruct`, coder-specialized) differ primarily in pretraining mix. The coder variant has seen large quantities of code where `"..."` single-line strings are common (code embedded in Python strings, markdown code blocks flattened to one line, etc.). Fine-tuning on single-line AIL activated that prior — the model sometimes thought "this one-line AIL code I'm about to emit is a string literal, so let me wrap it in quotes."

The non-coder variant hasn't been saturated with that pattern. It reads a single-line assistant response as plain content, not as an escape target. So the artifact simply doesn't appear.

## What this says about training format

Single-line training is **viable on non-coder bases**. That's the affirmative finding. But:

- v6 (non-coder + single-line) = 62% answer
- v3 (coder + indented) = 70% answer

There's still an 8pp gap favoring the indented-coder combination. This gap could be:
- Non-coder base having weaker programming ability overall (likely large part)
- Single-line format losing some structural signal the tokenizer uses (possible smaller part)

A clean next experiment, if desired, would be v7: non-coder + indented. If v7 ≈ v6, the gap is about the base model. If v7 > v6 closing the gap, the format does matter a bit.

## Decision

- **Serving model: v3 (unchanged).** v6 does not replace v3 because the overall answer rate is still 8pp lower.
- **v5 regression hypothesis closed.** R5's leading-quote artifact is definitively attributed to the coder-base Python prior. Single-line training is rehabilitated as a valid format for non-coder models.
- **v7 (non-coder + indented) is conditional.** Only worth running if we want to isolate "format" from "base model" as the remaining 8pp explainer. Not urgent.

## What about Python-side answer collapse (62% → 2%)?

v6's Python side drops to 2% answer (6/50 parsed, 1/50 answered correctly). Expected: v6 is a fine-tune of a general Qwen2.5-7B-Instruct specialized toward AIL. It was never going to write good Python. This column is not a meaningful signal for the AIL track; use the qwen14b or Sonnet baselines when the question is "same task, different language."

The grammar-level safety property holds regardless: AIL 0% error-handling miss, Python 50% error-handling miss. Language-invariant.
