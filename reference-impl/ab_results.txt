# Can Removing `while` Prevent Infinite Loops?

No, removing the `while` keyword alone cannot prevent infinite loops in a programming language.

The reason is that infinite loops are fundamentally about **unbounded recursion or iteration**, not about any specific keyword. Even without `while`, a language can still express infinite loops through:

1. **Recursive functions** - A function that calls itself without a proper termination condition creates an infinite loop, regardless of whether `while` exists
2. **Other looping constructs** - `for` loops can be infinite (e.g., `for(;;)` in C), and `goto` statements can create cycles
3. **Higher-order functions** - Constructs like `recursion schemes` or custom iterators can encode infinite behavior

In fact, according to the **Halting Problem** (proven by Alan Turing), any Turing-complete language—one powerful enough to perform general computation—cannot guarantee that all programs will terminate. The only way to truly prevent infinite loops is to restrict the language so severely that it loses Turing-completeness, typically by disallowing unbounded recursion entirely and only permitting loops with statically-determinable bounds (like primitive recursive functions). Simply removing `while` doesn't achieve this; you'd need to remove or restrict recursion and all other mechanisms for unbounded iteration.

However, if you remove `while` *and* restrict the language so that all loops must iterate over finite data structures (like how `for-each` loops work in functional languages), you can guarantee termination. This is because such loops can only iterate a bounded number of times determined by the size of the collection. Languages designed for total correctness sometimes enforce this by only allowing primitive recursion or iteration over provably finite structures, ensuring that every program must eventually halt—a property useful in proof assistants and systems where non-termination would be catastrophic.
